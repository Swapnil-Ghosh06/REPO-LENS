Help & Documentation

Why RepoLens?

A README tells you what a project does. RepoLens tells you how it does it, specific to your exact task. Skip the endless documentation hunt and instantly interrogate the architecture, locate key logic, and get actionable answers. It turns static code into a dynamic, interactive conversation.

Getting Started

Make sure your local RepoLens backend is running, then click the Index Repository button. We'll read the code, embed it intelligently, and get ready for your questions. This only takes a minute and only happens once per repo!

Good Questions to Ask

  • ✓ "How does authentication work in this repo?"
  • ✓ "Find all database queries and explain them"
  • ✓ "Explain the main entry point of the app"
  • ✓ "What does the UserService class do?"
  • ✓ "How is error handling implemented?"

Limitations & API Limits

  • • Public repositories only (v1)
  • • Maximum 500 files / 50MB per repo
  • • Uses Google Gemini's free tier (15 requests/min)

If you hit the AI rate limit, simply wait 60 seconds and try again. Brief pauses during indexing are completely normal to prevent overloading the API.

Troubleshooting

Service Worker "Inactive"?
If you open your browser's extension settings and see the service worker listed as "inactive," do not worry! This is a built-in feature of modern Chrome extensions to save memory. The service worker goes to sleep automatically and will wake up instantly the moment you interact with RepoLens.

Backend offline → Ensure your RepoLens server application is running.
Key: NONE error → Check that your configuration file has your API key set correctly.
Index fails → The repository may be too large (>500 files).
