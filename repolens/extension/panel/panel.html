RepoLens

Getting Started

The backend must be running before you use this extension.
Start it with: uvicorn main:app --reload
Run this from the repolens/backend/ folder with your venv active.

What is Indexing?

RepoLens reads every code file in the repository, breaks them into logical chunks, and stores them as searchable vectors. This takes 1-3 minutes for most repos and only needs to happen once per repo.

Good Questions to Ask

  • ✓ "How does authentication work in this repo?"
  • ✓ "Find all database queries"
  • ✓ "Explain the main entry point"
  • ✓ "What does the UserService class do?"
  • ✓ "How is error handling implemented?"

Limitations

  • Public repositories only (v1)
  • Maximum 500 files / 50MB per repo
  • Backend must be running on localhost:8000
  • Requires a free Gemini API key

API Limits

RepoLens uses Google Gemini's free tier.
Limit: 15 requests/minute, 1,500/day.
If you hit the limit, wait 60 seconds and try again.
Indexing large repos may pause briefly — this is normal.

Troubleshooting

Backend offline → Start uvicorn, check your terminal for errors
Key: NONE error → Check .env file has no spaces around the = sign
Index fails → Repo may be too large (>500 files). Try a smaller repo.
Answer is wrong → Re-index the repo, the codebase may have changed.
